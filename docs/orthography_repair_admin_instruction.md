# Как заполнять `orthography_repair`

## Поле `Targets`

Формат строки:

```txt
id | surface | replacement | type | options | occurrence
```

## Что значит каждое поле

`id` - уникальный id ошибки. Обычно достаточно `target_1`.

`surface` - ошибочный фрагмент ровно так, как он написан в тексте.

`replacement` - правильная замена.

`type` - тип фрагмента:

- `word` - одно слово;
- `span` - несколько слов.

`options` - варианты ответа через запятую. Обязательно добавьте правильный вариант.

`occurrence` - номер вхождения, если одинаковый фрагмент встречается в тексте несколько раз. Обычно можно оставить пустым.

## Пример для замены одного слова

```txt
target_1 | опозновательных | опознавательных | word | опозновательных, опознавательных, опозновательних, опазнавательных |
```

Поле `Repairs`:

```txt
target_1 | опознавательных
```

## Пример для замены нескольких слов

```txt
target_1 | в тоже время | в то же время | span | в тоже время, в то же время, втоже время, в тоже-время |
```

Поле `Repairs`:

```txt
target_1 | в то же время
```

## Важно

`surface` должен полностью совпадать с фрагментом в поле "Текст с ошибкой", иначе превью не сможет найти кликабельную область.
