Самый разумный следующий шаг:
1. **Аудит главной**

   - загрузка следующего упражнения или пула;
   - submit/check answer;
   - статистика/прогресс, если она приходит с сервера.

3. **Не трогать UI-state**  
   Историю чата, текущую анимацию, локальный прогресс, выбранные варианты лучше оставить в Zustand/React state. React Query должен отвечать за server state, не за всё подряд.

4. **Потом внедрить минимальный слой**
   Query keys, `useQuery`/`useMutation`, prefetch следующего упражнения, invalidate/patch прогресса после ответа.
